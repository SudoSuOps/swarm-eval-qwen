# Release Process

## When to Release

Block-0 becomes v1.0.0 when:
- All three eval suites pass acceptance criteria
- Regression gate passes against all prior phases
- Adversarial suite has no critical failures
- Tokenizer verified (P1 = P2 = P3 match)
- Provenance chain complete and clean

## Release Pack Contents

Generated by `scripts/07_release_pack.sh`:

```
release_pack/
    MODEL_CARD.md           Model card (architecture, training, eval)
    eval_report.json        Combined core/edges/adversarial metrics
    config_bundle/          Phase 1-3 configs + sampling
    provenance_bundle/      All provenance JSONs
    hashes.json             SHA256 of adapter + merged shards
    LICENSE
    KNOWN_LIMITATIONS.md    Safety notes + failure categories
```

## Versioning

| Version | Meaning |
|---------|---------|
| v1.0.0 | Block-0 sealed, all gates pass |
| v1.0.1 | Targeted patch (specific failure fix) |
| v1.1.0 | New eval suite added, re-evaluated |
| v2.0.0 | Block-1 (retrained on production data) |
